Introdução

Neste trabalho estão documentados diversos conceitos/comandos de SQL necessários para a manutenção de uma base de dados.

A estrutura consiste em vários cards que contém informação sobre cada comando: o que faz, a sua sintaxe e exemplos práticos da sua utilização. Estes exemplos serão realizados no Supabase, Sistema Gestor de Base de Dados (SGBD) relacional, cuja engine é PostgreSQL, outro SGBD relacional.

NOTA:

  • Não estão presentes todos os conceitos e comandos de SQL, apenas o necessário para a realização da disciplina de Implementação e Exploração de Bases de Dados;
  • Os exemplos podem mostrar falhas ao nível da normalização de dados.


ENQUADRAMENTO TEÓRICO

Structured Query Language

Structured Query Language, normalmente designada por SQL, é uma linguagem de programação concebida para gerir e manipular bases de dados relacionais. O seu desenvolvimento teve início na década de 1970 pela IBM. A consequente evolução atribuiu à linguagem um estatuto universal, tornando-se parte fundamental dos Sistemas Gestores de Bases de Dados (SGBD) e, portanto, utilizada em vários sectores para o tratamento de dados.

A linguagem SQL fornece um conjunto abrangente de comandos, ou queries (plural de query), para executar diversas tarefas, tais como: consultar, inserir, atualizar e apagar dados; criar e modificar esquemas de bases de dados e controlar acessos.

A universalidade do SQL deve-se à sua simplicidade e versatilidade na interação independente da plataforma, ou seja, permite trabalhar com diferentes SGBD's através do mesmo código.

GESTÃO DE TABELAS

CREATE TABLE

O comando CREATE TABLE é utilizado para criar uma tabela. A tabela pode ser criada como nova, identificando assim as colunas pertencentes e o respetivo tipo de dados, ou a partir de outra tabela já existente, através do comando CREATE TABLE SELECT.

SINTAXE → CREATE TABLE

CREATE TABLE [tabela] (
  [coluna] [tipo],
  [coluna] [tipo],
  [coluna] [tipo],
  ...
);

SINTAXE → CREATE TABLE + SELECT

CREATE TABLE [tabela] AS
SELECT [coluna], [coluna], ...
FROM [tabela_existente];
  • O SELECT funciona como uma consula normal, ou seja, pode ser usado * para selecionar todas as colunas;
  • Este comando apenas copia nome e tipo de dados, não copia restrições.
    •

    ALTER TABLE

    O comando ALTER TABLE permite alterar estrutura da tabela: criar/eliminar colunas (ADD e DROP COLUMN), alterar nomes (RENAME COLUMN) e tipos de dados das colunas (ALTER/MODIFY COLUMN). Este último exemplo pode variar a sintaxe consoante o SGBD utilizado.

    SINTAXE → ADD (COLUMN)

    ALTER TABLE [tabela]
    ADD [coluna] [tipo];

    SINTAXE → RENAME COLUMN

    ALTER TABLE [tabela]
    RENAME COLUMN [nome_antigo] TO [nome_novo];

    SINTAXE → ALTER/MODIFY COLUMN

    Access | SQL Server
    ALTER TABLE [tabela]
    ALTER COLUMN [coluna] [tipo_novo];
    MySQL
    ALTER TABLE [tabela]
    MODIFY COLUMN [coluna] TYPE [tipo_novo];
    Oracle (Versão 10+)
    ALTER TABLE [tabela]
    MODIFY ([coluna] [tipo_novo]);

    OBSERVAÇÃO: Em Oracle, não é estritamente necessária a utilização de parêntesis, embora seja uma boa prática.


    PostgreSQL
    ALTER TABLE [tabela]
    ALTER COLUMN [coluna] TYPE [tipo_novo];
  • As sintaxes em Access e SQL Server, MySQL e Oracle poderão não estar corretas.
  • A troca de tipos de dados apresentada é feita dentro da mesma categoria. Exemplificando, caso a coluna seja do tipo inteiro, só pode ser trocado diretamente para um tipo numérico.
    •

    SINTAXE → DROP COLUMN

    ALTER TABLE [tabela]
    DROP COLUMN [coluna];

    DROP TABLE

    O comando DROP TABLE elimina permanentemente uma tabela.

    SINTAXE → DROP TABLE

    DROP TABLE [tabela];

    PRIMARY KEY

    A PRIMARY KEY, ou chave primária, é um valor único e não nulo que identifica de forma única um registo de uma tabela. Cada tabela pode ter apenas uma chave primária, que pode ser simples ou composta, sendo constituída por uma ou várias colunas, respetivamente.

    SINTAXE → PRIMARY KEY + CREATE TABLE

    SINTAXE → CHAVE PRIMÁRIA SIMPLES

    CREATE TABLE [tabela] (
      [coluna_chave] [tipo] PRIMARY KEY,
      [coluna] [tipo],
      [coluna] [tipo],
      ...,
    );
    CREATE TABLE [tabela] (
      [coluna_chave] [tipo],
      [coluna] [tipo],
      [coluna] [tipo],
      ...
      PRIMARY KEY ([coluna_chave]),
    );

    OBSERVAÇÃO: Ambos os comandos retornam o mesmo resultado, diferenciando apenas o local da declaração da chave primária.

    SINTAXE → CHAVE PRIMÁRIA COMPOSTA

    CREATE TABLE [tabela] (
      [coluna_chave_1] [tipo],
      [coluna_chave_2] [tipo],
      [coluna] [tipo],
      ...,
      PRIMARY KEY ([coluna_chave_1], [coluna_chave_2]);
    );

    SINTAXE → CHAVE PRIMÁRIA COM NOME

    CREATE TABLE [tabela] (
      [coluna_chave] [tipo],
      [coluna] [tipo],
      [coluna] [tipo],
      ...,
      CONSTRAINT [nome_chave] PRIMARY KEY ([coluna_chave]);
    );

    OBSERVAÇÃO: Este comando pode ser usado para atribuir chaves primárias simples e compostas, basta colocar as colunas desejadas.

  • Sintaxe em Access, MySQL, SQL Server e Oracle não foi testada;
  • O nome padrão da chave primária muda consoante o SGBD. Exemplificando, para uma tabela aluno, em PostgreSQL o nome padrão é aluno_pkey, e em SQL Server e Oracle é aluno_PK.
    •

    SINTAXE → PRIMARY KEY + ALTER TABLE

    SINTAXE → CHAVE PRIMÁRIA SIMPLES

    ALTER TABLE [tabela]
    ADD PRIMARY KEY ([coluna_chave]);

    SINTAXE → CHAVE PRIMÁRIA COMPOSTA | CHAVE PRIMÁRIA COM NOME

    ALTER TABLE [tabela]
    ADD CONSTRAINT [nome_chave] PRIMARY KEY ([coluna_chave_1], [coluna_chave_2]);

    OBSERVAÇÃO: Este comando pode ser usado para atribuir chaves primárias simples e compostas, basta colocar as colunas desejadas.

  • Só pode existir uma única chave primária por tabela. Caso já exista, é recomendável eliminá-la ou então atribuir à coluna as constraints UNIQUE e NOT NULL;
  • Este comando não cria nova(s) coluna(s), portanto, é requisito as colunas mencionadas existirem na tabela.
    •

    SINTAXE → ELIMINAR CHAVE PRIMÁRIA

    Access | SQL Server | Oracle | PostgreSQL
    ALTER TABLE [tabela]
    DROP CONSTRAINT [nome_chave];
    MySQL
    ALTER TABLE [tabela]
    DROP PRIMARY KEY;
  • Sintaxe em Access, MySQL, SQL Server e Oracle não foi testada;
  • Este comando não elimina coluna(s), apenas apaga a constraint PRIMARY KEY.
    •

    FOREIGN KEY

    A FOREIGN KEY, ou chave estrangeira, é um valor que identifica, a partir de uma tabela, um registo de uma tabela relacionada. A chave estrangeira funciona também como protetora de relações, para que estas não sejam afetadas criticamente durante operações de base de dados. O tipo de dados da chave estrangeira deve coincidir com a coluna relacionada.

    Para além disso, a chave estrangeria é especialmente útil no processo de normalização de bases de dados.

    SINTAXE → FOREIGN KEY + CREATE TABLE

    SINTAXE → CHAVE ESTRANGEIRA

    CREATE TABLE [tabela] (
      [coluna_chave] [tipo] PRIMARY KEY,
      [coluna_chave_estrangeira] [tipo] FOREIGN KEY REFERENCES [tabela_relacionada]([coluna_unica]),
      [coluna_chave] [tipo],
      ...,
    );
    CREATE TABLE [tabela] (
      [coluna_chave] [tipo],
      [coluna_chave_estrangeira] [tipo],
      ...,
      PRIMARY KEY ([coluna_chave]),
      FOREIGN KEY ([coluna_chave_estrangeira]) REFERENCES [tabela_relacionada]([coluna_unica])
    );

    OBSERVAÇÃO: Ambos os comandos retornam o mesmo resultado, diferenciando apenas o local da declaração da chave estrangeira.

    SINTAXE → CHAVE ESTRANGEIRA COM NOME

    CREATE TABLE [tabela] (
      [coluna_chave] [tipo],
      [coluna_chave_estrangeira] [tipo],
      [coluna_chave_estrangeira_2] [tipo],
      ...,
      PRIMARY KEY ([coluna_chave])
      CONSTRAINT [nome_chave] FOREIGN KEY ([coluna_chave_estrangeira]) REFERENCES [tabela_relacionada]([coluna_unica]),
    );

    SINTAXE → FOREIGN KEY + ALTER TABLE

    SINTAXE → CHAVE ESTRANGEIRA

    ALTER TABLE [tabela]
    ADD FOREIGN ([coluna_chave_estrangeira]) REFERENCES [tabela_relacionada]([coluna_unica]);

    OBSERVAÇÃO: Podem ser atribuídas múltiplas chaves estrangeiras, repetindo a expressão ADD.

    SINTAXE → CHAVE ESTRANGEIRA COM NOME

    ALTER TABLE [tabela]
    ADD CONSTRAINT [nome_chave] FOREIGN KEY ([coluna_chave_estrangeira]) REFERENCES [tabela_relacionada]([coluna_unica]);
  • Este comando não cria nova(s) coluna(s), portanto, é requisito as colunas mencionadas existirem na tabela.
    •

    SINTAXE → ELIMINAR CHAVE ESTRANGEIRA

    Access | SQL Server | Oracle | PostgreSQL
    ALTER TABLE [tabela]
    DROP CONSTRAINT [nome_chave];
    MySQL
    ALTER TABLE [tabela]
    DROP FOREIGN KEY [nome_chave];
  • Sintaxe em Access, MySQL, SQL Server e Oracle não foi testada;
  • Este comando não elimina coluna(s), apenas apaga a constraint FOREIGN KEY.
    •

    CONSTRAINTS

    Constraints são restrições que podem ser aplicadas às colunas de uma tabela, durante a sua criação ou alteração. Em caso de alteração de tabelas, mais concretamente as colunas, não são criadas/eliminas colunas, mas sim as suas restrições.

    OBSERVAÇÃO: As constraints PRIMARY KEY, FOREIGN KEY e CREATE INDEX são exploradas mais a fundo nas categorias Chaves, para as duas primeiras, e Índices, para a última.

    CONSTRAINT NOT NULL

    A constraint NOT NULL impede a coluna de assumir o valor nulo (NULL).

    SINTAXE → NOT NULL + CREATE TABLE

    CREATE TABLE [tabela] (
      [coluna] [tipo] NOT NULL,
      ...
    );

    SINTAXE → NOT NULL + ALTER TABLE

    Access | SQL Server
    ALTER TABLE [tabela]
    ALTER COLUMN [coluna] [tipo] NOT NULL;
    MySQL
    ALTER TABLE [tabela]
    MODIFY COLUMN [coluna] [tipo] NOT NULL;
    Oracle (Versão 10+)
    ALTER TABLE [tabela]
    MODIFY [coluna] [tipo] NOT NULL;
    PostgreSQL
    ALTER TABLE [tabela]
    ALTER COLUMN [coluna] SET NOT NULL;
  • Excluindo o PostgreSQL, a sintaxe apresentada pode ser utilizada para remover a constraint NOT NULL, omitindo essa mesma expressão, já que o comando reescreve a coluna;
  • Em PostgreSQL, para eliminar a constraint NOT NULL, basta substituir a palavra-chave SET por DROP.
    •

    CONSTRAINT UNIQUE

    A constraint UNIQUE impede a coluna de assumir valores repetivos, tornando-a única.

    SINTAXE → UNIQUE + CREATE TABLE

    SINTAXE → UNIQUE SIMPLES

    CREATE TABLE [tabela] (
      [coluna_chave] [tipo] UNIQUE,
      ...
    );
    CREATE TABLE [tabela] (
      [coluna_chave] [tipo],
      ...
      UNIQUE ([coluna_chave]),
    );

    OBSERVAÇÃO: Ambos os comandos retornam o mesmo resultado, diferenciando apenas o local da declaração da constraint UNIQUE.

    SINTAXE → UNIQUE COMPOSTO | UNIQUE COM NOME

    CREATE TABLE [tabela] (
      [coluna_chave_1] [tipo],
      [coluna_chave_2] [tipo],
      ...
      CONSTRAINT [nome_chave] UNIQUE ([coluna_chave_1], [coluna_chave_2])
    );

    OBSERVAÇÃO: Ao atribuir uma constraint UNIQUE composta será criada uma restrição na qual a combinação das colunas selecionadas deve ser única.

    SINTAXE → UNIQUE + ALTER TABLE

    SINTAXE → UNIQUE SIMPLES

    Access
    ALTER TABLE [tabela]
    ADD CONSTRAINT [nome_restrição] UNIQUE ([coluna]);
    MySQL | SQL Server | Oracle | PostgreSQL
    ALTER TABLE [tabela]
    ADD UNIQUE ([coluna]);

    SINTAXE → UNIQUE COMPOSTO | UNIQUE COM NOME

    ALTER TABLE [tabela]
    ADD CONSTRAINT [nome_chave] UNIQUE ([coluna_chave_1], [coluna_chave_2]);

    SINTAXE → ELIMINAR UNIQUE

    Access | SQL Server | Oracle | PostgreSQL
    ALTER TABLE [tabela]
    DROP CONSTRAINT [nome_chave];
    MySQL
    ALTER TABLE [tabela]
    DROP INDEX [nome_chave];

    CONSTRAINT CHECK

    A constraint CHECK assegura que o valor de uma coluna segue uma determinada condição.

    SINTAXE → CHECK + CREATE TABLE

    SINTAXE → CHECK SIMPLES

    CREATE TABLE [tabela] (
      [coluna] [tipo] CHECK ([condição]),
      ...
    );
    CREATE TABLE [tabela] (
      [coluna] [tipo],
      ...,
      CHECK ([condição])
    );

    OBSERVAÇÃO: Ambos os comandos retornam o mesmo resultado, diferenciando apenas o local da declaração da constraint CHECK.

    SINTAXE → CHECK COMPOSTO | CHECK COM NOME

    CREATE TABLE [tabela] (
      [coluna] [tipo],
      ...,
      CONSTRAINT [nome_chave] CHECK ([condição] [operador] [condição])
    );

    SINTAXE → CHECK + ALTER TABLE

    SINTAXE → CHECK SIMPLES

    ALTER TABLE [tabela]
    ADD CHECK ([condição]);

    SINTAXE → CHECK COMPOSTO | CHECK COM NOME

    ALTER TABLE [tabela]
    ADD CONSTRAINT [nome_chave] CHECK ([condição_1] [operador] [condição_2]);

    SINTAXE → ELIMINAR CHECK

    Access | SQL Server | Oracle | PostgreSQL
    ALTER TABLE [tabela]
    DROP CONSTRAINT [nome_chave];
    MySQL
    ALTER TABLE [tabela]
    DROP CHECK [nome_chave];

    CONSTRAINT DEFAULT

    A constraint DEFAULT aufere à coluna um valor padrão.

    SINTAXE → DEFAULT + CREATE TABLE

    SINTAXE → DEFAULT

    CREATE TABLE [tabela] (
      [coluna] [tipo] DEFAULT [valor_padrão],
      ...
    );

    SINTAXE → DEFAULT + ALTER TABLE

    SINTAXE → DEFAULT

    Access | MySQL | PostgreSQL
    ALTER TABLE [tabela]
    ALTER COLUMN [coluna] SET DEFAULT [valor_padrão];
    MySQL
    ALTER TABLE [tabela]
    ALTER [coluna] SET DEFAULT [valor_padrão];
    SQL Server
    ALTER TABLE [tabela]
    ADD CONSTRAINT [nome_chave]
    DEFAULT [valor_padrão] FOR [coluna];
    Oracle
    ALTER TABLE [tabela]
    MODIFY [coluna] DEFAULT [valor_padrão];

    SINTAXE → ELIMINAR DEFAULT

    Access | SQL Server | Oracle | PostgreSQL
    ALTER TABLE [tabela]
    ALTER COLUMN [coluna] DROP DEFAULT;
    MySQL
    ALTER TABLE [tabela]
    ALTER [coluna] DROP DEFAULT;

    GESTÃO E CONSULTA DE DADOS

    INSERT INTO

    O comando INSERT INTO insere um ou mais novos registos numa tabela. Estes registos podem ser novos ou então cópias de uma outra tabela, cujos tipos de dados devem corresponder. Para transferir os dados entre tabelas, é utilizado o comando INSERT INTO SELECT.

    SINTAXE → INSERT INTO

    INSERT INTO [tabela] ([coluna], [coluna], ...) VALUES
    ([valor], [valor], ...);
  • As colunas podem ser ordenadas aleatóriamente, desde que as colunas, os valores e os respetivos tipos correspondam devidamente;
  • Os nomes das colunas da tabela podem ser omitidos caso a sua ordem corresponda à das colunas da tabela, ou seja, caso na tabela a ordem seja id, nome, idade, os dados a inserir devem seguir esta mesma ordem;
  • Podem ser inseridos mais do que um registo de cada vez. Basta repetir a expressão ([valor], [valor], ...) e separar essas diferentes instâncias por vírgulas.
    •

    SINTAXE → INSERT INTO + SELECT

    INSERT INTO [tabela] ([coluna], [coluna], ...)
    SELECT [coluna], [coluna], ...
    FROM [tabela_existente]
    WHERE [condição];
  • Em caso de consulta a todas as colunas, os nomes podem ser substituídos por *;
  • A condição pode ser omitida, inserindo todos os dados das colunas selecionadas.
    •

    SELECT

    O comando SELECT seleciona registos de uma determinada tabela. Normalmente é acompanhado por uma condição, que faz a filtragem dos registos, com recurso à cláusula WHERE e os diferentes operadores. Pode ser também incluída a palavra-chave DISTINCT, que faz com que apenas sejam selecionados os diferentes valores de uma coluna.

    SINTAXE → SELECT

    SELECT [coluna], [coluna], ...
    FROM [tabela]
    WHERE [condição];
  • Em caso de consulta a todas as colunas, os nomes podem ser substituídos por *;
  • A condição pode ser omitida, inserindo todos os dados das colunas selecionadas.
    •

    SINTAXE → SELECT + DISTINCT

    Consulta de todas as colunas:

    SELECT DISTINCT [coluna], [coluna], ...
    FROM [tabela];

    OBSERVAÇÃO: Ao selecionar duas ou mais tabelas, o comando SELECT DISTINCT mostrará todas as diferentes combinações dessas colunas existentes na tabela.

    UPDATE

    O comando UPDATE altera um ou mais registos de uma determinada tabela.

    SINTAXE → UPDATE

    UPDATE [tabela]
    SET [coluna] = [valor], [coluna] = [valor], ...
    WHERE [condição];

    OBSERVAÇÃO: Ao omitir a condição, o comando UPDATE irá alterar todos os registos da tabela.

    DELETE

    O comando DELETE elimina permanentemente um ou mais registos de uma determinada tabela.

    SINTAXE → DELETE

    DELETE FROM [tabela]
    WHERE [condição];

    OBSERVAÇÃO: Ao omitir a condição, o comando DELETE irá eliminar todos os registos da tabela.

    TRUNCATE TABLE

    O comando TRUNCATE TABLE elimina todos os registos de uma tabela. Este comando faz o mesmo que o DELETE sem condição.

    SINTAXE → TRUNCATE

    TRUNCATE TABLE [tabela];

    CLÁUSULAS

    Cláusulas são palavras-chave que especificam determinadas definições de uma consulta, tais como especificação de registos, ordenação, agrupamentos, entre outras.

    OBSERVAÇÃO: A sua inclusão no comando SQL não é obrigatória.

    CLÁUSULA WHERE

    A cláusula WHERE atribui uma condição à consulta, filtrando os registos da tabela, com recurso a operadores. Esta cláusula é utilizada, portanto, em consultas, mas também nos comandos UPDATE e DELETE para selecionar o(s) registo(s) a ser(em) alterado(s) ou eliminado(s).

    SINTAXE → WHERE + SELECT

    SELECT [coluna], [coluna], ...
    FROM [tabela]
    WHERE [condição];

    SINTAXE → WHERE + UPDATE

    UPDATE [tabela]
    SET [coluna] = [valor], [coluna] = [valor], ...
    WHERE [condição];

    SINTAXE → WHERE + DELETE

    DELETE FROM [tabela]
    WHERE [condição];

    CLÁUSULA ORDER BY

    A cláusula ORDER BY ordena os registos consultados de uma tabela, de forma ascendente/crescente, descendente/decrescente ou ambas. Esta cláusula, numa consulta, posiciona-se à frente da cláusula WHERE.

    SINTAXE → ORDER BY + ASC (UMA OU MÚLTIPLAS COLUNAS)

    SELECT [coluna], [coluna], ...
    FROM [tabela]
    WHERE [condição]
    ORDER BY [coluna] ASC, [coluna] ASC, ...;

    OBSERVAÇÃO: A palavra-chave ASC pode ser omitida.

    SINTAXE → ORDER BY + DESC (UMA OU MÚLTIPLAS COLUNAS)

    SELECT [coluna], [coluna], ...
    FROM [tabela]
    WHERE [condição]
    ORDER BY [coluna] DESC, [coluna] DESC, ...;

    SINTAXE → ORDER BY + ASC + DESC (UMA OU MÚLTIPLAS COLUNAS)

    SELECT [coluna], [coluna], ...
    FROM [tabela]
    WHERE [condição]
    ORDER BY [coluna] ASC, [coluna] DESC, ...;
  • A palavra-chave ASC pode ser omitida;
  • A ordenação é feita consoante a ordem das colunas selecionadas.
    •

    CLÁUSULA TOP

    A cláusula TOP (SQL Server/Access), LIMIT (MySQL/PostgreSQL) ou FETCH FIRST (Oracle) cria um limite de resultados a ser mostrados durante uma consulta.

    SINTAXE → TOP/LIMIT

    Access | SQL Server
    SELECT TOP [número/percentagem] [coluna], [coluna], ...
    FROM [tabela]
    WHERE [condição];
    ORDER BY [coluna], [coluna], ...;
    MySQL | PostgreSQL
    SELECT TOP [número/percentagem] [coluna], [coluna], ...
    FROM [tabela]
    WHERE [condição];
    ORDER BY [coluna], [coluna], ...
    LIMIT [número];
    Oracle (Versão 12+)
    SELECT [coluna], [coluna], ...
    FROM [tabela]
    WHERE [condição];
    ORDER BY [coluna], [coluna], ...
    FETCH FIRST [número] ROWS ONLY;

    OPERADORES

    Os operadores são símbolos/palavras utilizadas para comparar duas ou mais expressões. São utilizadas em conjunto com a cláusula WHERE.

    OBSERVAÇÃO: Não serão abordados os operadores mais comuns: '=', '<>' ou '!=', '>', '≥', '<', '≤'.

    OPERADORES AND, OR, NOT

    Os operadores AND, OR e NOT são os mais simples da linguagem SQL. AND e OR fazem a comparação entre duas condições diferentes. Já NOT mostra o resultado contrário da condição presente.

    SINTAXE → AND

    WHERE [condição] AND [condição];

    SINTAXE → OR

    WHERE [condição] OR [condição];

    SINTAXE → NOT

    WHERE NOT [condição];

    SINTAXE → AND + OR

    WHERE [condição] AND [condição] OR [condição];

    OBSERVAÇÃO: A utilização ou não de parêntesis influencia o resultado, já que AND tem prioridade em relação ao OR.

    SINTAXE → NOT + AND

    WHERE NOT ([condição] AND [condição]);

    SINTAXE → NOT + OR

    WHERE NOT ([condição] OR [condição]);

    SINTAXE → NOT + BETWEEN | NOT + IN | NOT + LIKE

    BETWEEN
    WHERE [coluna] NOT BETWEEN [valor] AND [valor];
    IN
    WHERE [coluna] NOT IN ([valor], [valor], ...);
    LIKE
    WHERE [coluna] NOT LIKE '[padrão]';

    OPERADOR BETWEEN

    O operador BETWEEN filtra registos de uma determinada tabela consoante um intervalo de valores. O operador é inclusivo, ou seja, os limites também contam.

    SINTAXE → BETWEEN

    WHERE [coluna] BETWEEN [valor] AND [valor];

    OPERADOR IN

    O operador IN filtra registos de uma determinada tabela consoante um determinado conjunto de valores alfanuméricos específicos.

    SINTAXE → IN

    WHERE [coluna] IN ([valor], [valor], ...);

    OPERADOR LIKE

    O operador LIKE filtra registos de uma determinada tabela de acordo com um padrão alfanumérico. Este padrão faz recurso às wildcards.

    SINTAXE → LIKE

    WHERE [coluna] LIKE '[padrão]';

    FUNÇÕES

    As funções, em SQL, são métodos que realizam cálculos acerca de colunas e registos consultados.

    FUNÇÃO MIN

    A função MIN() retorna o valor mínimo de um determinado campo numérico.

    SINTAXE → MIN()

    SELECT MIN([coluna])

    OBSERVAÇÃO: Em colunas de string, a função MIN() retorna a primeira string alfabeticamente ordenada.

    FUNÇÃO MAX

    A função MAX() retorna o valor máximo de um determinado campo numérico.

    SINTAXE → MAX()

    SELECT MAX([coluna])

    OBSERVAÇÃO: Em colunas de string, a função MAX() retorna a última string alfabeticamente ordenada.

    FUNÇÃO SUM

    A função SUM() retorna a soma dos valores de um determinado campo numérico.

    SINTAXE → SUM()

    SELECT SUM([coluna])

    OBSERVAÇÃO: Em colunas de string, a função SUM() pode ser utilizada caso todos os valores sejam convertíveis para um valor numérico. Esta funcionalidade não ocorre em todos os SGBD.

    FUNÇÃO AVG

    A função AVG() retorna a média dos valores de um determinado campo numérico.

    SINTAXE → AVG()

    SELECT AVG([coluna])

    OBSERVAÇÃO: Em colunas de string, a função AVG() pode ser utilizada caso todos os valores sejam convertíveis para um valor numérico. Esta funcionalidade não ocorre em todos os SGBD.

    FUNÇÃO COUNT

    A função COUNT() retorna o número de registos ou número de valores não nulos de uma coluna. Para contar o número de valores diferentes, é utilizada a palavra-chave DISTINCT.

    SINTAXE → COUNT()

    SELECT COUNT([coluna]);

    OBSERVAÇÃO: Nome das colunas pode ser substituído por *, retornando número total de registos.

    SINTAXE → COUNT() + DISTINCT

    SELECT COUNT(DISTINCT [coluna]);

    ALIAS

    Alias é uma funcionalidade em SQL que permite atribuir um apelido a uma tabela ou coluna. Este apelido é dado através da palavra-chave AS.

    Alias → Tabela

    SELECT [apelido_1].[coluna], [apelido_2].[coluna], ...
    FROM [tabela] AS [apelido_1], [tabela_2] AS [apelido_2]
    WHERE [condição];
  • O apelido pode conter espaços desde que esteja entre parêntesis retos ('[]');
  • A palavra-chave AS pode ser ignorada, quando a tabela é chamada apenas uma vez, sendo necessária em caso contrário, transformando a sintaxe em [tabela] [apelido].
    •

    Alias → Coluna

    SELECT [coluna] AS [apelido], [coluna] AS [apelido], ...
    FROM [tabela]
    WHERE [condição];
  • O apelido pode conter espaços desde que esteja entre parêntesis retos ('[]');
  • A palavra-chave AS pode ser ignorada, transformando a sintaxe em [coluna] [apelido].

    WILDCARDS

    Wildcards são carateres que substituem um ou mais carateres numa string. Os wildcards são utilizados em conjunto com o operador LIKE para criar padrões de filtragem de registos.

    OBSERVAÇÃO: Os padrões utilizados, normalmente, são case insensitive, sendo necessária a verificação da documentação do SGBD a ser utilizado.

    Wildcard → '%'

    O widlcard '%' simboliza zero ou vários carateres, e tem caráter universal, já que é suportado por todos os SGBD.

    Em Access, '%' é substituído pelo carater '*'.

    Exemplificando, 'A%' traduz-se para todas as strings que comecem por 'A', sendo seguidas por uma indeterminada sequência de carateres.

    Wildcard → '_'

    O widlcard '_' simboliza um único carater e, assim como '%', este carater é suportado por todos os SGBD.

    Em Access, '_' é substituído pelo carater '?'.

    Exemplificando, '_a%' traduz-se para todas as strings que comecem por um determinado carater, seguido de um 'a' e uma indeterminada sequência de carateres.

    Wildcard → '[]'

    O widlcard '[]' simboliza qualquer carater dentro dos parêntesis retos. Este wildcard não é universal, pois não é suportado pelo PostgreSQL e MySQL.

    Exemplificando, '[ACD]%' traduz-se para todas as strings que comecem por um 'A', 'C' ou 'D', seguidos de uma indeterminada sequência de carateres.

    Wildcard → '^'

    O widlcard '^' simboliza qualquer carater fora dos parêntesis retos. Este wildcard não é universal, pois não é suportado pelo PostgreSQL e MySQL.

    Em Access, '^' é substituído pelo carater '!'.

    Exemplificando, '[^ACD]%' traduz-se para todas as strings que comecem não comecem por um 'A', 'C' ou 'D', seguidos de uma indeterminada sequência de carateres.

    Wildcard → '-'

    O widlcard '-' simboliza qualquer carater dentro do limite definido. Este wildcard não é universal, pois não é suportado pelo PostgreSQL e MySQL.

    Exemplificando, '[A-F]%' traduz-se para todas as strings que comecem comecem por um 'A', 'B', 'C', 'D', 'E', 'F', seguidos de uma indeterminada sequência de carateres.

    AUXILIARES DE CONSULTA

    INNER JOIN

    O comando INNER JOIN seleciona todos os registos que têm valores comuns entre duas ou mais tabelas.

    SINTAXE → INNER JOIN

    SELECT [tabela].[coluna], [tabela_relacionada].[coluna], ...
    FROM [tabela]
    INNER JOIN [tabela_relacionada] ON [tabela].[coluna_relacionada] = [tabela_relacionada].[coluna_relacionada];

    LEFT JOIN

    O comando LEFT JOIN seleciona todos os registos da tabela da esquerda e os registos que têm valores comuns da tabela da direita. Caso não haja correspondência, é retornado valor NULL.

    OBSERVAÇÃO: Em alguns casos, este comando escreve-se LEFT OUTER JOIN.

    SINTAXE → LEFT JOIN

    SELECT [tabela_esquerda].[coluna], [tabela_direita].[coluna], ...
    FROM [tabela_esquerda]
    LEFT JOIN [tabela_direita] ON [tabela_esquerda].[coluna_relacionada] = [tabela_direita].[coluna_relacionada];

    RIGHT JOIN

    O comando RIGHT JOIN seleciona todos os registos da tabela da direita e os registos que têm valores comuns da tabela da esquerda. Caso não haja correspondência, é retornado valor NULL

    OBSERVAÇÃO: Em alguns casos, este comando escreve-se RIGHT OUTER JOIN.

    SINTAXE → RIGHT JOIN

    SELECT [tabela_esquerda].[coluna], [tabela_direita].[coluna], ...
    FROM [tabela_esquerda]
    RIGHT JOIN [tabela_direita] ON [tabela_esquerda].[coluna_relacionada] = [tabela_direita].[coluna_relacionada];

    FULL JOIN

    O comando FULL JOIN seleciona todos os registos, quer tenham valores correspondentes entre tabelas ou não.

    OBSERVAÇÃO: Em alguns casos, este comando escreve-se FULL OUTER JOIN.

    SINTAXE → FULL JOIN

    SELECT [tabela_esquerda].[coluna], [tabela_direita].[coluna], ...
    FROM [tabela_esquerda]
    FULL OUTER JOIN [tabela_direita] ON [tabela_esquerda].[coluna_relacionada] = [tabela_direita].[coluna_relacionada];

    CROSS JOIN

    O comando CROSS JOIN retorna todas as combinações de registos entre duas ou mais tabelas.

    SINTAXE → CROSS JOIN

    SELECT [tabela].[coluna], [tabela_relacionada].[coluna], ...
    FROM [tabela]
    CROSS JOIN [tabela_relacionada];

    ÍNDICE

    O índice é um objeto que serve de referência a uma chave, tendo como principal objetivo a otimização das consultas à base de dados.

    O índice de uma base de dados serve como um índice de um livro. Para encontrar um conteúdo desejado, já que folhear todo o livro custa muito tempo, recorremos ao índice para descobrir em que página se encontra essa informação. O índice numa base de dados faz o mesmo, em vez de procurar informação percorrendo toda a tabela, o SGBD recorre a uma referência direta aos registos desejados.

    Pesquisar um determinado registo pela chave primária, que é única, é fácil e rápido. Mas procurar registos través de um campo variável, como nome, já exige outro raciocínio. Sem um índice, o SGBD terá de percorrer toda a tabela à procura do nome desejado. Mas, com um índice criado para a coluna nome, o SGBD através de referências, vai diretamente aos registos com o nome a pesquisar, o que economiza tempo, principalmente favorável em bases de dados com grande dimensão.

    A criação de índices é positiva, mas o seu exagero pode causar ainda mais problemas durante a consulta, pois o SGBD terá de testar qual o melhor índice para pesquisar determinados campos, e esta escolha pode não ser a melhor.

    Assim, em suma, a criação de índices é útil para campos que são utilizamos como filtros, como nome, idade, curso, e evitar de criar demasiados, pois pode atrasar ainda mais o processo de consulta.

    CREATE INDEX

    O comando CREATE INDEX cria um índice. Este pode ser simples, composto ou único.

    SINTAXE → CREATE INDEX

    Índice simples

    CREATE INDEX [índice]
    ON [tabela] ([coluna]);

    Índice composto

    CREATE INDEX [índice]
    ON [tabela] ([coluna], [coluna], ...);

    NOTA:

    • As colunas do índice composto devem constituir um conjunto de colunas utilizado para filtar registos, como por exemplo filtrar alunos por ano e curso ao mesmo tempo.


    Índice único

    CREATE UNIQUE INDEX [índice]
    ON [tabela] ([coluna]);

    NOTA:

    • A coluna do índice único não pode conter valores repetidos, como por exemplo email.

    DROP INDEX

    O comando DROP INDEX elimina permanentemente um índice.

    SINTAXE → DROP INDEX

    DROP INDEX [nome];
